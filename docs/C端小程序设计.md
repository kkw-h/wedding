太棒了。移动端小程序（C端）是直接面向**新人（新郎新娘）**的。

如果说后台管理系统（Admin）是理性的、严谨的“中央厨房”，那么小程序就是端给客人的**“精美摆盘”**。它的设计目标不是“管理”，而是**“陪伴”与“减压”**。

**核心设计关键词：**
*   **情绪价值 (Emotional):** 界面要浪漫、温馨，缓解备婚焦虑。
*   **游戏化 (Gamified):** 把繁琐的任务变成“打怪升级”。
*   **极简操作 (Minimalist):** 能点一下解决的，绝不让用户输入文字。

# 📱 移动端小程序 UI 设计方案 (v1.0)

## 1. 全局导航结构 (Navigation)

采用经典的 **底部 Tab Bar (Bottom Navigation)** 结构，共 5 个标签：

1.  **首页 (Home):** 仪表盘、倒计时、核心通知。
2.  **任务 (Tasks):** 备婚清单、进度打卡。
3.  **方案 (Plan):** 报价单、设计图、支付。
4.  **工具 (Tools):** 电子请柬、宾客名单、排桌图。
5.  **我的 (Mine):** 个人信息、服务团队、合同文件。

---

## 2. 详细页面 UI 设计

### Tab 1: 首页 (Home) - “浪漫的指挥舱”

**视觉风格：** 大留白、高品质背景图（婚纱照或意向图）、衬线字体（Serif）。

*   **顶部 Hero 区：**
    *   **背景：** 动态的磨砂玻璃效果，透出底部的婚礼主色调。
    *   **内容：**
        *   `张伟 & 王芳` (艺术字体)
        *   `距离婚礼还有 128 天` (大号数字，带呼吸动效)。
        *   `当前阶段：方案设计期` (胶囊标签)。
*   **中部：智能动态卡片 (Smart Feed)：**
    *   *逻辑：* 根据当前状态只显示最重要的 1-2 件事。
    *   *卡片样式：*
        *   📢 **待确认：** “策划师刚刚上传了《主舞台设计稿v2》，请查收。” [去查看]
        *   💰 **待支付：** “中期款 ¥30,000 需在 3 天内支付。” [去支付]
*   **底部：服务团队 (My Team)：**
    *   横向滑动的圆形头像列表：主策划、设计师、摄影师、化妆师。
    *   *交互：* 点击头像直接弹出“拨打电话”或“复制微信”。

### Tab 2: 任务 (Tasks) - “备婚大冒险”

**设计理念：** 消除清单的压迫感，用**“闯关”**的形式呈现。

*   **顶部：进度总览**
    *   环形进度条：`已完成 35%`。
    *   文案激励：“太棒了！击败了全国 80% 的新人。”
*   **主体：时间轴卡片流 (Timeline Cards)**
    *   **分组：** 按月份或阶段（如：倒计时6个月）。
    *   **任务卡片 UI：**
        *   左侧：图标（如 👠 代表婚鞋，💌 代表请柬）。
        *   中间：任务标题 + 截止日期（如：`选购婚鞋` | `10月5日截止`）。
        *   右侧：状态按钮。
            *   ⚪ 空心圆：未开始。
            *   🟡 锁定锁头：前置未完成。
            *   🟢 实心对钩：已完成。
*   **任务详情页 (点击卡片进入)：**
    *   **攻略区：** “不知道怎么选婚鞋？看看这篇指南 👇” (内嵌图文)。
    *   **操作区：**
        *   如果是“购买类”：提供 [上传凭证照片] 按钮。
        *   如果是“确认类”：提供 [确认无误] 按钮。
    *   **底部评论：** 类似微信朋友圈评论，新人可以和策划师在这里针对该任务留言。

### Tab 3: 方案与预算 (Plan & Budget) - “所见即所得”

这是最容易产生纠纷的地方，UI 必须清晰、透明。

*   **顶部切换 (Segment Control):** `设计方案` | `报价预算` | `支付记录`。
*   **子页面 A：设计方案 (The Vision)**
    *   **瀑布流布局：** 展示策划师上传的效果图、手绘图。
    *   **交互：** 点击图片全屏查看，支持**“图片打点评论”**（类似飞书/Figma）。
        *   *场景：* 新娘点一下图片上的花艺，输入：“这个粉色太深了，换浅一点”。
*   **子页面 B：报价预算 (The Money)**
    *   **隐私保护：** 默认隐藏总价，点击“小眼睛”图标显示。
    *   **分类折叠列表：**
        *   `> 场地布置 (¥30,000)` [展开]
        *   `> 人员配置 (¥15,000)` [展开]
    *   **明细行 UI：**
        *   左图右文。
        *   `香槟色玫瑰` x `200支` = `¥xxx`。
        *   *关键交互：* 如果策划师修改了价格，该行高亮显示 `[已修改]` 标签，点击可查看变更历史。

### Tab 4: 工具 (Tools) - “实用百宝箱”

*   **九宫格图标入口：**
    *   💌 **电子请柬：** H5 制作器，选模板 -> 换照片 -> 分享微信。
    *   🪑 **排桌图：**
        *   *移动端特化交互：* 既然手机屏幕小，不要做复杂的拖拽。
        *   *逻辑：* 列表式管理。点击“第1桌”，从通讯录勾选 10 个人塞进去。
    *   📝 **宾客统计：** 关联电子请柬的回执，自动统计“出席人数”、“忌口”。
    *   🎁 **礼金记账：** 婚礼当天的记账本。

### Tab 5: 我的 (Mine)

*   **卡片式布局：**
    *   **合同中心：** 查看电子签名的 PDF 合同。
    *   **账号绑定：**
        *   *伴侣绑定：* 邀请另一半加入，数据实时同步（双人备婚）。
        *   *父母账号：* (可选) 开启“只读模式”给父母看进程。
    *   **设置：** 消息推送开关。

## 3. 关键交互动效 (Micro-interactions)

为了提升“高级感”和“愉悦感”，这几个动效必须做：

1.  **撒花庆祝 (Confetti):**
    *   当用户在“任务”页点击“完成”时，屏幕必须爆出彩带/花瓣动画，并伴随轻微的震动反馈 (Haptic Feedback)。
2.  **平滑展开 (Smooth Expand):**
    *   查看报价单明细时，点击分类，列表要有丝滑的下拉动画，而不是生硬的跳转。
3.  **加载占位 (Skeleton Screen):**
    *   在加载高清设计图时，不要显示转圈圈，要显示灰色的骨架屏，保持界面稳定性。

## 4. 特殊模式：策划师视图 (Planner Mode via Mini-program)

策划师虽然主要用 PC，但周末在婚礼现场时也需要用小程序。
系统通过**账号角色判断**，如果是策划师登录，底部 Tab 变为：

1.  **工作台：** 今日所有婚礼的日程表（精确到分钟）。
2.  **消息：** 聚合所有客户的留言。
3.  **现场执行：**
    *   **物料核对单：** 现场收货勾选。
    *   **人员签到：** 生成二维码，让兼职人员扫码签到（关联自动算工资）。

## 5. 技术实现建议 (Tech Stack)

*   **框架：** Uni-app 或 Taro (一次开发，同时发布微信小程序 + 抖音小程序)。
*   **UI 组件库：** 推荐使用 **Vant Weapp** 或 **TDesign (腾讯系)**，风格现代且稳定。
*   **实时通讯：** WebSocket (用于双人同步状态，比如新郎勾选了任务，新娘手机上立刻变绿)。
